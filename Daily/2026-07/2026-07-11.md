## SQL

### 오늘 푼 문제

1. 과일로 만든 아이스크림 고르기
2. 조건에 맞는 도서 리스트 출력하기

### 배운 점

- `JOIN`을 사용하면 두 테이블에서 연관된 데이터를 함께 조회할 수 있다.
- `ON`은 두 테이블을 어떤 컬럼 기준으로 연결할지 지정한다.
- `WHERE`에서 여러 조건을 함께 사용할 때 `AND`를 사용할 수 있다.
- 날짜 조건을 걸 때 `BETWEEN`을 사용할 수 있다.
- `ORDER BY`로 조회 결과를 정렬할 수 있다.

### 사용한 SQL

```sql
SELECT FIRST_HALF.FLAVOR
FROM FIRST_HALF
JOIN ICECREAM_INFO
ON FIRST_HALF.FLAVOR = ICECREAM_INFO.FLAVOR
WHERE TOTAL_ORDER > 3000 AND INGREDIENT_TYPE = 'fruit_based'
ORDER BY FIRST_HALF.TOTAL_ORDER DESC;
```

```sql
SELECT BOOK_ID, PUBLISHED_DATE
FROM BOOK
WHERE CATEGORY = '인문'
AND PUBLISHED_DATE BETWEEN '2021-01-01' AND '2021-12-30'
ORDER BY PUBLISHED_DATE;
```

## Network

### 오늘 들은 강의

혼자 공부하는 네트워크 4강 ~ 7강

### 핵심 키워드

- 패킷: 네트워크에서 데이터를 주고받는 단위
- 헤더: 송수신지 주소 같은 부가 정보가 담기는 영역
- 프로토콜: 노드 간 정보를 올바르게 주고받기 위한 규칙
- OSI 7계층: 네트워크 통신 과정을 7단계로 나눈 참조 모델
- TCP/IP 4계층: 실제 인터넷 통신에서 많이 사용하는 네트워크 계층 모델
- 캡슐화: 송신 과정에서 계층을 내려가며 헤더나 트레일러를 붙이는 과정
- 역캡슐화: 수신 과정에서 계층을 올라가며 헤더나 트레일러를 확인하고 제거하는 과정

### 배운 점

네트워크에서 메시지는 한 번에 그대로 전송되는 것이 아니라 패킷 단위로 나뉘어 이동한다.

각 계층은 자신의 역할에 맞는 프로토콜을 사용하고, 송신 과정에서는 캡슐화가 일어나며 수신 과정에서는 역캡슐화가 일어난다.

### 스키마와 데이터베이스

스키마는 서로 연관된 데이터들을 그룹핑하는 단위이다.

MySQL에서는 `CREATE DATABASE` 명령어로 데이터베이스를 생성할 수 있다.

여러 스키마를 저장하고 관리하는 시스템이 데이터베이스 서버이다.

### 데이터베이스 서버의 장점

데이터베이스 서버는 보안 체계를 제공한다.

사용자별로 권한을 부여할 수 있어서 데이터를 안전하게 관리할 수 있다.

### MySQL 접속

```bash
cd /usr/local/mysql/bin
./mysql -uroot -p
```

### 실습한 SQL

```sql
CREATE DATABASE opentutorials;
```

`opentutorials`라는 데이터베이스를 생성한다.

```sql
DROP DATABASE opentutorials;
```

`opentutorials` 데이터베이스를 삭제한다.

```sql
SHOW DATABASES;
```

현재 MySQL 서버에 존재하는 데이터베이스 목록을 조회한다.

```sql
USE opentutorials;
```

앞으로 사용할 데이터베이스를 `opentutorials`로 선택한다.

### 오늘 헷갈린 점

```sql
SHOW DATABASE opentutorials;
```

위 명령어는 잘못된 문법이었다.

데이터베이스 목록을 확인할 때는 `SHOW DATABASES;`를 사용하고, 특정 데이터베이스를 선택할 때는 `USE 데이터베이스명;`을 사용한다.
