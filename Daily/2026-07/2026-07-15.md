## Database

### 오늘 공부한 내용

데이터베이스 개론 6강 ~ 10강을 들으며 관계형 데이터베이스의 구성요소, 키, 정규화, 반정규화, SQL 분류를 공부했다.

### 관계형 데이터베이스 구성요소

- 테이블: 데이터를 행과 열로 저장하는 2차원 구조
- 속성: 테이블의 열, 데이터의 특성을 나타냄
- 튜플: 테이블의 행, 하나의 데이터 묶음
- 디그리: 속성의 수
- 카디널리티: 튜플의 수

### 키의 종류

- 후보키: 유일성과 최소성을 만족하는 키
- 기본키: 후보키 중 대표로 선택한 키
- 대체키: 기본키로 선택되지 않은 후보키
- 슈퍼키: 튜플을 유일하게 식별할 수 있는 키
- 외래키: 다른 테이블의 기본키를 참조하는 키

### 정규화

정규화는 데이터 중복으로 인해 발생할 수 있는 삽입 이상, 삭제 이상, 갱신 이상을 줄이기 위해 테이블을 분리하는 과정이다.

정규화를 통해 데이터 중복을 줄이고 무결성을 높일 수 있다.

### 반정규화

반정규화는 성능 향상을 위해 정규화된 테이블을 의도적으로 통합하거나 중복을 허용하는 작업이다.

정규화는 무결성에 유리하지만 조인이 많아지고 쿼리가 복잡해질 수 있다.

### SQL 분류

- DDL: 데이터 구조를 정의
- DML: 데이터를 조회, 삽입, 수정, 삭제
- DCL: 권한과 보안을 제어
- TCL: 트랜잭션을 제어

## MySQL

### 오늘 공부한 내용

오늘 MySQL에서는 관계형 데이터베이스가 필요한 이유와 `JOIN`을 공부했다.

데이터베이스 개론에서 공부한 정규화, 외래키, 테이블 분리 개념과 MySQL 실습 내용이 연결되었다.

### 관계형 데이터베이스와 JOIN

관계형 데이터베이스는 중복된 데이터를 줄이고, 관련 있는 데이터를 여러 테이블로 나누어 관리하기 위해 사용한다.

테이블을 나누면 중복은 줄어들지만 데이터를 한눈에 보기 어려워질 수 있다.

이때 `JOIN`을 사용하면 나누어진 테이블을 다시 연결해서 조회할 수 있다.

### 실습한 SQL

```sql
SELECT *
FROM topic
LEFT JOIN author
ON topic.author_id = author.id;
```

`SELECT *`을 사용하면 두 테이블의 모든 컬럼이 출력되어 결과가 복잡해질 수 있다.

테이블 별칭과 필요한 컬럼만 선택하면 결과를 더 깔끔하게 볼 수 있다.

```sql
SELECT
    t.id,
    t.title,
    t.description,
    t.created,
    a.name,
    a.profile
FROM topic t
LEFT JOIN author a
ON t.author_id = a.id;
```

### DB 개론과 연결

DB 개론에서 정규화는 중복을 줄이기 위해 테이블을 나누는 과정이라고 배웠다.

MySQL 실습에서는 나누어진 `topic` 테이블과 `author` 테이블을 `JOIN`으로 다시 연결해 조회했다.

## SQL

### 오늘 푼 문제

1. 강원도에 위치한 생산공장 목록 출력하기
2. 서울에 위치한 식당 목록 출력하기

### 배운 점

- `LIKE`는 문자열이 특정 패턴과 일치하는지 검사할 때 사용한다.
- `%`는 글자 수와 상관없이 아무 문자열이나 올 수 있다는 의미이다.
- 집계 함수를 사용할 때 일반 컬럼과 함께 조회하려면 `GROUP BY`가 필요하다.
- 전체 데이터에 대한 집계 값 하나만 구하는 경우에는 `GROUP BY`가 필요하지 않다.

### 사용한 SQL

```sql
SELECT FACTORY_ID, FACTORY_NAME, ADDRESS
FROM FOOD_FACTORY
WHERE ADDRESS LIKE '강원도%';
```

```sql
SELECT
    I.REST_ID,
    I.REST_NAME,
    I.FOOD_TYPE,
    I.FAVORITES,
    I.ADDRESS,
    ROUND(AVG(R.REVIEW_SCORE), 2) AS SCORE
FROM REST_INFO I
JOIN REST_REVIEW R
ON I.REST_ID = R.REST_ID
WHERE ADDRESS LIKE '서울%'
GROUP BY I.REST_ID
ORDER BY SCORE DESC, I.FAVORITES DESC;
```

- 싱글 모드 광섬유

### 배운 점

NIC는 호스트와 통신 매체를 연결하는 네트워크 장비이다.

호스트가 네트워크를 통해 송수신하는 정보는 NIC를 거치며, NIC는 MAC 주소를 인식하고 오류가 있는 프레임을 폐기할 수 있다.

트위스티드 페어 케이블은 구리선으로 전기 신호를 주고받는 통신 매체이고, 광섬유 케이블은 빛을 이용해 정보를 주고받는 통신 매체이다.

광섬유 케이블은 코어의 지름에 따라 싱글 모드와 멀티 모드로 나눌 수 있다.
