## SQL

### 오늘 푼 문제

1. 12세 이하인 여자 환자 목록 출력하기
2. 흉부외과 또는 일반외과 의사 목록 출력하기

### 배운 점

- `IFNULL()`을 사용하면 `NULL` 값을 원하는 값으로 바꿔서 조회할 수 있다.
- `WHERE`에서 `AND`를 사용하면 여러 조건을 모두 만족하는 데이터를 조회할 수 있다.
- `WHERE`에서 `OR`를 사용하면 여러 조건 중 하나라도 만족하는 데이터를 조회할 수 있다.
- `ORDER BY`에서 여러 컬럼을 기준으로 정렬할 수 있다.
- `DESC`는 내림차순, `ASC`는 오름차순 정렬이다.

### 사용한 SQL

```sql
SELECT
    PT_NAME,
    PT_NO,
    GEND_CD,
    AGE,
    IFNULL(TLNO, 'NONE') AS TLNO
FROM PATIENT
WHERE AGE <= 12 AND GEND_CD = 'W'
ORDER BY AGE DESC, PT_NAME ASC;
```

```sql
SELECT DR_NAME, DR_ID, MCDP_CD, HIRE_YMD
FROM DOCTOR
WHERE MCDP_CD = 'CS' OR MCDP_CD = 'GS'
ORDER BY HIRE_YMD DESC, DR_NAME ASC;
```

## Network

### 오늘 들은 강의

혼자 공부하는 네트워크 8강 ~ 10강

### 핵심 키워드

- 네트워크 참조 모델
- 프로토콜
- 네트워크 장비
- 트래픽
- 처리율
- 대역폭
- 패킷 손실

### 배운 점

네트워크 참조 모델은 네트워크를 이해하기 위한 기준이지만, 실제 네트워크를 작동시키는 주체는 프로토콜과 네트워크 장비이다.

OSI 7계층과 TCP/IP 4계층은 시험이나 면접을 위해서는 중요하지만, 실무 관점에서는 각 계층에서 동작하는 프로토콜과 장비를 이해하는 것이 더 중요하다.

트래픽은 네트워크 내에서 오가는 정보량이다. 한 노드가 처리할 수 있는 트래픽에는 한계가 있기 때문에 트래픽이 과도하게 몰리면 성능 저하가 발생할 수 있다.

네트워크 성능은 처리율, 대역폭, 패킷 손실 같은 지표로 판단할 수 있다.

### 용어 정리

- 처리율: 단위 시간당 네트워크를 통해 실제로 전송되는 정보량
- 대역폭: 단위 시간 동안 통신 매체를 통해 송수신할 수 있는 최대 정보량
- 패킷 손실: 송수신되는 패킷이 중간에 유실되는 상황

## Database / MySQL

### 오늘 한 공부

MySQL 강의를 이어서 듣고, `CREATE TABLE`로 테이블을 생성하는 방법을 공부했다.

MySQL은 데이터를 저장하고 관리하기 위한 관계형 데이터베이스 관리 시스템이다.

데이터베이스 안에 실제 데이터를 저장하려면 먼저 테이블을 만들어야 한다.

테이블을 만들 때는 컬럼 이름, 데이터 타입, `NULL` 허용 여부, 기본키 등을 함께 정의한다.

### 실습한 SQL

```sql
CREATE TABLE topic (
    id INT(11) NOT NULL AUTO_INCREMENT,
    title VARCHAR(100) NOT NULL,
    description TEXT NULL,
    created DATETIME NOT NULL,
    author VARCHAR(30) NULL,
    profile VARCHAR(100) NULL,
    PRIMARY KEY (id)
);
```

### 배운 점

- `CREATE TABLE`은 새로운 테이블을 생성할 때 사용한다.
- `AUTO_INCREMENT`는 숫자 값을 자동으로 증가시킨다.
- `PRIMARY KEY`는 각 행을 고유하게 구분하는 기본키이다.
- `VARCHAR`는 짧은 문자열, `TEXT`는 긴 문자열을 저장할 때 사용할 수 있다.
- `DATETIME`은 날짜와 시간을 저장하는 타입이다.

### 다음에 정리할 내용
