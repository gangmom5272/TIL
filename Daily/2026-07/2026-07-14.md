## Database

### 오늘 공부한 내용

데이터베이스 개론 1강 ~ 5강을 들으며 데이터베이스의 기본 개념과 DBMS의 역할을 공부했다.

### 핵심 개념

- 데이터: 가공되지 않은 그대로의 객관적인 사실
- 정보: 목적에 따라 데이터를 처리하여 의미를 부여한 것
- 데이터베이스: 특정 목적을 위해 관련 있는 데이터들을 구조적으로 모아 놓은 저장소
- DBMS: 사용자와 데이터베이스 사이에서 데이터베이스를 관리해주는 프로그램
- 스키마: 데이터베이스의 구조와 제약 조건에 대한 명세
- 메타데이터: 데이터를 설명하기 위한 데이터
- 데이터 사전: 데이터베이스에 사용되는 데이터 구조와 메타데이터를 저장하고 관리하는 곳

### 데이터 분류

- 정형 데이터: 구조가 정해진 데이터
- 반정형 데이터: XML, JSON처럼 내부 정보를 표현하는 메타데이터가 포함된 데이터
- 비정형 데이터: 영상, 이미지, 텍스트 문서처럼 고정된 구조로 표현하기 어려운 데이터

### DBMS의 필수 기능

- 정의 기능: 데이터의 타입, 구조, 이용 방식, 제약 조건 등을 정의
- 조작 기능: 데이터의 검색, 삽입, 삭제, 갱신 등을 처리
- 제어 기능: 데이터 무결성 유지, 권한 검사, 병행 제어 등을 수행

### 데이터베이스 언어

- DDL: 데이터베이스의 구조를 정의하는 언어
- DML: 데이터를 조회, 삽입, 수정, 삭제하는 언어
- DCL: 데이터 무결성, 보안, 권한, 회복, 병행 제어를 수행하는 언어

### 데이터베이스 설계

데이터베이스 설계는 개념적 설계, 논리적 설계, 물리적 설계 단계로 나뉜다.

- 개념적 설계: 현실 세계를 추상화하여 E-R 다이어그램으로 표현
- 논리적 설계: DBMS가 이해할 수 있는 논리적 자료 구조로 변환
- 물리적 설계: 실제 저장장치에 저장될 물리적 구조로 변환

## MySQL

### 오늘 공부한 내용

생활코딩 MySQL 강의에서 CRUD를 공부하고 직접 실습했다.

CRUD는 데이터를 다루는 기본 기능을 의미한다.

- Create: 데이터 추가
- Read: 데이터 조회
- Update: 데이터 수정
- Delete: 데이터 삭제

### Create

```sql
INSERT INTO 테이블명 (컬럼1, 컬럼2)
VALUES (데이터1, 데이터2);
```

### Read

```sql
SELECT 컬럼명
FROM 테이블명;
```

전체 컬럼을 조회할 때는 `*`를 사용할 수 있다.

```sql
SELECT *
FROM 테이블명;
```

조건을 걸어 조회할 때는 `WHERE`를 사용한다.

```sql
SELECT *
FROM 테이블명
WHERE 조건;
```

정렬할 때는 `ORDER BY`, 조회 개수를 제한할 때는 `LIMIT`을 사용할 수 있다.

```sql
SELECT *
FROM 테이블명
ORDER BY 컬럼명 DESC
LIMIT 10;
```

### Update

```sql
UPDATE 테이블명
SET 컬럼명 = '수정사항'
WHERE 조건;
```

`WHERE` 조건을 작성하지 않으면 해당 컬럼의 모든 데이터가 수정될 수 있으므로 주의해야 한다.

### Delete

```sql
DELETE FROM 테이블명
WHERE 조건;
```

`DELETE`도 `WHERE` 조건을 작성하지 않으면 테이블의 모든 데이터가 삭제될 수 있으므로 주의해야 한다.

### DB 개론과 연결

DB 개론에서 배운 DML은 데이터를 조회, 삽입, 수정, 삭제하는 언어이다.

MySQL에서 실습한 `INSERT`, `SELECT`, `UPDATE`, `DELETE`가 DML에 해당한다.

## SQL

### 오늘 푼 문제

1. 3월에 태어난 여성 회원 목록 출력하기
2. 조건에 부합하는 중고거래 댓글 조회하기

### 배운 점

- `DATE_FORMAT()`을 사용하면 날짜 데이터를 원하는 형식으로 출력할 수 있다.
- `MONTH()`를 사용하면 날짜 컬럼에서 월만 추출해 조건을 걸 수 있다.
- `JOIN`을 사용할 때 테이블 별칭을 붙이면 쿼리를 더 짧고 읽기 쉽게 작성할 수 있다.
- 두 테이블에 같은 이름의 컬럼이 있거나 테이블명이 길 때 별칭을 사용하면 가독성이 좋아진다.

### 사용한 SQL

```sql
SELECT
    MEMBER_ID,
    MEMBER_NAME,
    GENDER,
    DATE_FORMAT(DATE_OF_BIRTH, '%Y-%m-%d') AS DATE_OF_BIRTH
FROM MEMBER_PROFILE
WHERE MONTH(DATE_OF_BIRTH) = 3
AND GENDER = 'W'
AND TLNO IS NOT NULL;
```

```sql
SELECT
    USED_GOODS_BOARD.TITLE,
    USED_GOODS_BOARD.BOARD_ID,
    USED_GOODS_REPLY.REPLY_ID,
    USED_GOODS_REPLY.WRITER_ID,
    USED_GOODS_REPLY.CONTENTS,
    DATE_FORMAT(USED_GOODS_REPLY.CREATED_DATE, '%Y-%m-%d') AS CREATED_DATE
FROM USED_GOODS_BOARD
JOIN USED_GOODS_REPLY
ON USED_GOODS_BOARD.BOARD_ID = USED_GOODS_REPLY.BOARD_ID
WHERE YEAR(USED_GOODS_BOARD.CREATED_DATE) = 2022
AND MONTH(USED_GOODS_BOARD.CREATED_DATE) = 10
ORDER BY USED_GOODS_REPLY.CREATED_DATE ASC, USED_GOODS_BOARD.TITLE ASC;
```

## Network

### 오늘 들은 강의

혼자 공부하는 네트워크 11강 ~ 12강

### 핵심 키워드

- 이더넷
- IEEE 802.3
- 이더넷 프레임
- MAC 주소
- 프리앰블
- 타입/길이
- FCS
- 토큰 링

### 배운 점

이더넷은 현대 LAN, 특히 유선 LAN 환경에서 가장 대중적으로 사용되는 기술이다.

물리 계층에서는 케이블 같은 통신 매체의 규격을 다루고, 데이터 링크 계층에서는 이더넷 프레임의 형식을 다룬다.

MAC 주소는 같은 LAN 환경에서 데이터를 정확한 장치로 보내기 위해 사용하는 물리적 주소이다.

FCS는 수신한 이더넷 프레임에 오류가 있는지 확인하기 위한 오류 검출용 정보이다.
