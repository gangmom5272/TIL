## Data Engineering

### 오늘 읽은 내용

견고한 데이터 엔지니어링에서 데이터 엔지니어가 실무에서 누구와 함께 일하는지 읽었다.

데이터 엔지니어는 데이터를 생성하는 업스트림 직군과 데이터를 사용하는 다운스트림 직군 사이에서 데이터가 안정적으로 흐르도록 돕는 역할을 한다.

## 업스트림

업스트림은 데이터가 생성되는 영역이다.

데이터 엔지니어는 업스트림에서 만들어지는 데이터가 수집과 활용에 적합한 형태가 되도록 협업한다.

### 업스트림 주요 직군

- 백엔드 엔지니어
- 프론트엔드 엔지니어
- 모바일 앱 개발자
- 서비스 개발자
- 인프라 엔지니어
- DevOps 엔지니어
- 제품 관리자
- 서비스 운영 담당자

### 업스트림과의 협업

백엔드 엔지니어와는 서비스 데이터베이스, API, 이벤트 로그 설계에 대해 협업한다.

프론트엔드나 모바일 개발자와는 사용자 행동 로그를 어떤 시점에 어떤 형식으로 남길지 협의한다.

인프라나 DevOps 엔지니어와는 로그 수집, 배포 환경, 모니터링, 장애 대응 방식에 대해 협업한다.

제품 관리자와는 어떤 사용자 행동이나 서비스 지표를 데이터로 남겨야 하는지 논의한다.

## 다운스트림

다운스트림은 수집되고 처리된 데이터를 실제로 사용하는 영역이다.

데이터 엔지니어는 다운스트림 사용자가 필요한 데이터를 신뢰할 수 있는 형태로 사용할 수 있도록 제공한다.

### 다운스트림 주요 직군

- 데이터 분석가
- 데이터 사이언티스트
- 머신러닝 엔지니어
- AI 엔지니어
- 비즈니스 분석가
- 마케팅 담당자
- 기획자
- 경영진
- 제품 관리자

### 다운스트림과의 협업

데이터 분석가와는 분석에 필요한 테이블, 지표, 데이터 마트, 대시보드용 데이터 구조를 협의한다.

데이터 사이언티스트와 머신러닝 엔지니어와는 모델 학습에 필요한 데이터셋, 피처, 데이터 품질, 데이터 갱신 주기를 협의한다.

비즈니스 분석가나 마케팅 담당자와는 매출, 전환율, 사용자 행동, 캠페인 성과 같은 비즈니스 지표를 제공하기 위해 협업한다.

기획자와 제품 관리자와는 서비스 개선에 필요한 지표와 실험 결과를 확인할 수 있도록 데이터를 제공한다.

경영진에게는 의사결정에 필요한 핵심 지표가 정확하고 일관되게 전달되도록 지원한다.

## 오늘 배운 점

데이터 엔지니어는 업스트림과 다운스트림 사이에서 데이터 흐름을 연결하는 역할을 한다.

업스트림에서는 데이터가 올바르게 생성되고 수집될 수 있도록 협업하고, 다운스트림에서는 데이터를 사용하는 사람들이 신뢰할 수 있는 데이터를 사용할 수 있도록 제공한다.

데이터 엔지니어링은 단순히 데이터를 옮기는 일이 아니라, 조직 안에서 데이터가 생성되고 활용되는 전체 흐름을 설계하고 운영하는 일이다.
