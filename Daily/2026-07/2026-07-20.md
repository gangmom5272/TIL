### 데이터 엔지니어링

데이터 엔지니어링은 데이터를 수집, 저장, 처리, 제공할 수 있는 기반을 설계하고 운영하는 활동이다.

데이터가 분석, 서비스, 의사결정에 활용될 수 있도록 데이터 흐름을 만드는 일이 핵심이다.

### 데이터 엔지니어

데이터 엔지니어는 데이터가 필요한 곳에 안정적으로 도달할 수 있도록 데이터 파이프라인과 저장 구조를 설계하고 관리하는 역할을 한다.

단순히 데이터를 옮기는 사람이 아니라, 데이터의 품질, 신뢰성, 활용 가능성을 높이는 역할을 한다.

### 데이터 엔지니어링 수명 주기

데이터 엔지니어링 수명 주기는 데이터가 발생해서 최종적으로 활용되기까지의 전체 흐름을 의미한다.

데이터 생성, 수집, 저장, 변환, 제공, 활용 같은 단계로 이해할 수 있다.

### 데이터 성숙도

데이터 성숙도는 조직이 데이터를 얼마나 잘 수집, 관리, 활용하고 있는지를 나타내는 수준이다.

데이터 성숙도가 낮은 조직에서는 기본적인 데이터 수집과 저장 기반이 중요하고, 성숙도가 높은 조직에서는 데이터 품질, 자동화, 거버넌스, 확장성 같은 요소가 더 중요해진다.

## SQL

### 오늘 푼 문제

1. 오프라인/온라인 판매 데이터 통합하기
2. 역순 정렬하기

### 새로 알게 된 것

처음에는 두 테이블을 함께 조회해야 해서 `JOIN`만 생각했다.

하지만 오프라인/온라인 판매 데이터 통합 문제는 두 테이블을 공통 컬럼 기준으로 가로로 연결하는 문제가 아니라, 같은 구조의 판매 내역을 세로로 합치는 문제였다.

이럴 때는 `JOIN`이 아니라 `UNION` 또는 `UNION ALL`을 사용할 수 있다.

### JOIN과 UNION의 차이

- `JOIN`: 두 테이블을 공통 컬럼 기준으로 가로로 연결한다.
- `UNION`: 두 조회 결과를 세로로 합친다.
- `UNION ALL`: 중복 제거 없이 두 조회 결과를 그대로 세로로 합친다.

### 사용한 SQL

```sql
SELECT
    DATE_FORMAT(SALES_DATE, '%Y-%m-%d') AS SALES_DATE,
    PRODUCT_ID,
    USER_ID,
    SALES_AMOUNT
FROM ONLINE_SALE
WHERE SALES_DATE >= '2022-03-01'
AND SALES_DATE < '2022-04-01'

UNION ALL

SELECT
    DATE_FORMAT(SALES_DATE, '%Y-%m-%d') AS SALES_DATE,
    PRODUCT_ID,
    NULL AS USER_ID,
    SALES_AMOUNT
FROM OFFLINE_SALE
WHERE SALES_DATE >= '2022-03-01'
AND SALES_DATE < '2022-04-01'

ORDER BY SALES_DATE ASC, PRODUCT_ID ASC, USER_ID ASC;
```

오프라인 판매 데이터에는 `USER_ID`가 없기 때문에 `NULL AS USER_ID`로 컬럼 자리를 맞췄다.

`UNION ALL`을 사용할 때는 각 `SELECT`문의 컬럼 개수와 컬럼 순서가 같아야 한다.

```sql
SELECT NAME, DATETIME
FROM ANIMAL_INS
ORDER BY ANIMAL_ID DESC;
```

`역순 정렬하기` 문제는 `ORDER BY`와 `DESC`를 사용해 어렵지 않게 풀었다.

## MySQL Practice

### 오늘 실습한 내용

온라인 판매 데이터와 오프라인 판매 데이터를 각각 다른 테이블에 저장한 뒤, `UNION ALL`로 하나의 판매 내역처럼 조회했다.

### 실습 목표

- 온라인/오프라인 판매 데이터를 별도 테이블로 관리한다.
- 2022년 3월 판매 데이터만 조회한다.
- 두 테이블의 조회 결과를 `UNION ALL`로 합친다.
- 오프라인 판매 데이터에는 `user_id`가 없으므로 `NULL AS user_id`로 컬럼 구조를 맞춘다.

### 최종 성공 쿼리

```sql
SELECT sales_date, product_id, user_id, sales_amount
FROM online_sale
WHERE sales_date >= '2022-03-01'
AND sales_date < '2022-04-01'

UNION ALL

SELECT sales_date, product_id, NULL AS user_id, sales_amount
FROM offline_sale
WHERE sales_date >= '2022-03-01'
AND sales_date < '2022-04-01'

ORDER BY sales_date ASC, product_id ASC, user_id ASC;
```

### 배운 점

두 테이블을 합쳐야 한다고 해서 항상 `JOIN`을 사용하는 것은 아니다.

공통 컬럼을 기준으로 컬럼을 가로로 붙이면 `JOIN`을 사용하고, 같은 구조의 결과를 세로로 이어 붙이면 `UNION ALL`을 사용한다.

`UNION ALL`을 사용할 때는 두 `SELECT`문의 컬럼 개수와 순서를 맞춰야 한다.

컬럼이 없는 경우에는 `NULL AS 컬럼명`을 사용해서 구조를 맞출 수 있다.

### 배운 점

물리 계층에는 주소 개념이 없고, 데이터 링크 계층에는 MAC 주소 같은 주소 개념이 있다.

허브는 물리 계층 장비라 받은 신호를 판단하지 않고 모든 포트로 내보낸다.

허브는 반이중 통신을 사용하기 때문에 여러 호스트가 동시에 송신하면 충돌이 발생할 수 있다.

CSMA/CD는 반이중 이더넷 네트워크에서 충돌을 방지하기 위한 프로토콜이다.
