## Database

### 오늘 공부한 내용

데이터베이스 개론을 끝까지 듣고 정리했다.

오늘은 관계대수와 관계해석, 트랜잭션, 병행 제어, 장애와 회복, 분산 데이터베이스, 데이터베이스 보안을 공부했다.

### 관계대수와 관계해석

관계대수는 데이터베이스에서 원하는 정보를 검색하기 위해 절차적으로 연산을 명시하는 SQL의 이론적 기반이다.

관계해석은 원하는 결과가 무엇인지 논리적으로 표현하는 비절차적 언어이다.

정리하면 관계대수는 SQL의 실행 기반, 관계해석은 SQL의 표현 기반이고, SQL은 이를 결합한 실용 언어라고 이해할 수 있다.

### 트랜잭션

트랜잭션은 하나의 논리적 기능을 수행하기 위한 작업의 단위이다.

하나의 트랜잭션은 모두 성공하거나, 하나라도 실패하면 전체가 취소되어야 한다.

### ACID

- 원자성: 트랜잭션 내 연산은 모두 수행되거나 전부 취소되어야 한다.
- 일관성: 트랜잭션 수행 전과 후에도 데이터베이스는 일관된 상태를 유지해야 한다.
- 독립성: 동시에 실행되는 트랜잭션들은 서로 영향을 주지 않는 것처럼 실행되어야 한다.
- 영속성: 성공적으로 완료된 트랜잭션의 결과는 장애가 발생해도 영구적으로 반영되어야 한다.

### 병행 수행의 문제점

- 갱신 손실: 두 트랜잭션이 같은 데이터를 동시에 수정할 때 한 결과가 사라지는 현상
- 더티 리드: 아직 커밋되지 않은 데이터를 다른 트랜잭션이 읽는 현상
- 반복 불가능 읽기: 같은 데이터를 두 번 조회했는데 결과가 달라지는 현상
- 팬텀 리드: 같은 조건으로 조회했는데 조회되는 행의 개수가 달라지는 현상

### 회복

장애가 발생하면 데이터베이스는 Redo와 Undo를 통해 복구할 수 있다.

- Redo: 트랜잭션의 작업을 다시 실행하여 데이터베이스에 반영
- Undo: 실패한 트랜잭션의 작업을 취소하여 이전 상태로 복구

### 분산 데이터베이스

분산 데이터베이스는 여러 물리적인 데이터베이스를 네트워크로 연결해 하나의 논리적 데이터베이스처럼 운영하는 시스템이다.

분산 데이터베이스는 지역 자치성, 확장성, 가용성 측면에서 장점이 있지만, 설계와 운영이 복잡해질 수 있다.

### 데이터베이스 보안

데이터베이스 보안은 권한이 없는 사용자의 접근을 차단하고 데이터의 기밀성, 무결성, 가용성을 유지하기 위한 조치이다.

- 기밀성: 허락된 사용자만 데이터에 접근 가능
- 무결성: 데이터가 일관되고 정확한 상태를 유지
- 가용성: 허락된 사용자가 필요할 때 데이터에 접근 가능

## MySQL

### 오늘 공부한 내용

인터넷과 데이터베이스를 클라이언트/서버 구조로 이해했다.

인터넷에서는 클라이언트가 요청을 보내고 서버가 응답한다.

데이터베이스도 비슷하게 실제 데이터는 데이터베이스 서버에 저장되고, 데이터베이스 클라이언트가 명령을 보내면 서버가 그 명령에 따라 데이터를 제공한다.

### MySQL 클라이언트

지금까지 터미널에서 사용한 `mysql` 명령어 기반 도구는 MySQL Monitor이며, MySQL 클라이언트에 해당한다.

MySQL Workbench도 MySQL 클라이언트이지만, GUI로 데이터베이스를 조작할 수 있는 도구이다.

### CLI로 공부하는 이유

MySQL Workbench는 GUI로 쉽게 사용할 수 있지만, 당분간은 CLI 기반 MySQL Monitor로 공부하려고 한다.

CLI에서 직접 SQL 명령어를 입력하면 SQL 문법과 데이터베이스 동작 흐름을 더 명확하게 익힐 수 있다.

## SQL

### 오늘 푼 문제

1. 재구매가 일어난 상품과 회원 리스트 구하기
2. 모든 레코드 조회하기

### 배운 점

`GROUP BY`를 사용할 때는 “무엇을 기준으로 묶어서 집계할 것인가?”를 먼저 정해야 한다.

예를 들어 회원별 주문 수를 구하고 싶다면 회원을 기준으로 묶어야 하므로 `GROUP BY USER_ID`를 사용한다.

상품별 구매 수를 구하고 싶다면 상품을 기준으로 묶어야 하므로 `GROUP BY PRODUCT_ID`를 사용한다.

오늘 문제에서는 같은 회원이 같은 상품을 2번 이상 구매했는지 확인해야 했기 때문에 `USER_ID`, `PRODUCT_ID`를 함께 기준으로 묶었다.

### 사용한 SQL

```sql
SELECT USER_ID, PRODUCT_ID
FROM ONLINE_SALE
GROUP BY USER_ID, PRODUCT_ID
HAVING COUNT(*) >= 2
ORDER BY USER_ID ASC, PRODUCT_ID DESC;
```

```sql
SELECT *
FROM ANIMAL_INS
ORDER BY ANIMAL_ID;
```

### 오늘 이해한 것

집계 함수만 보는 것이 아니라, 결과를 어떤 단위로 나누어야 하는지를 먼저 생각해야 한다.

- 회원별 개수: `GROUP BY USER_ID`
- 상품별 개수: `GROUP BY PRODUCT_ID`
- 회원과 상품 조합별 개수: `GROUP BY USER_ID, PRODUCT_ID`
