# 혼자 공부하는 네트워크 11강 ~ 12강

## 이더넷

이더넷(Ethernet)은 현대 LAN, 특히 유선 LAN 환경에서 가장 대중적으로 사용되는 기술이다.

이더넷은 다양한 통신 매체의 규격, 송수신되는 프레임의 형태, 프레임을 주고받는 방법 등을 정의한 기술이다.

유선 LAN 환경은 대부분 이더넷을 기반으로 구성된다.

즉, 물리 계층에서 사용되는 케이블은 이더넷 규격을 따르고, 데이터 링크 계층에서 주고받는 프레임은 이더넷 형식을 따른다.

## 국제 표준으로서의 이더넷

이더넷은 IEEE 802.3이라는 이름으로 국제 표준이 되었다.

IEEE 802.3은 이더넷 관련 다양한 국제 표준의 모음이다.

오늘날에도 이더넷은 계속 발전하고 있으며 새로운 표준도 생기고 있다.

또한 IEEE 802.3은 이더넷 관련 표준을 만드는 작업 그룹을 의미하기도 한다.

## 물리 계층의 이더넷

이더넷 표준에 따라 통신 매체의 종류와 속도가 달라진다.

케이블을 표기할 때는 보통 다음 형식을 사용한다.

```text
전송속도BASE-추가특성
```

예시:

- 1000BASE-SX
- 5GBASE-T

### 전송 속도

숫자만 표기되어 있으면 Mbps 단위이고, 숫자 뒤에 G가 붙으면 Gbps 단위이다.

예시:

- 100BASE-T: 100Mbps 속도 지원
- 10GBASE-T: 10Gbps 속도 지원

### BASE

BASE는 베이스밴드(BASEband)의 약자이다.

BASE는 변조 타입을 의미한다.

변조 타입은 비트 신호로 변환된 데이터를 통신 매체로 전송하는 방법이다.

### 추가 특성

추가 특성에는 통신 매체의 특성이 명시될 수 있다.

예시:

- 전송 가능한 최대 거리
- 물리 계층 인코딩 방식
- 레인 수
- 통신 매체의 종류

### 통신 매체의 종류

| 표기 | 의미 |
|---|---|
| C | 동축 케이블 |
| T | 트위스티드 페어 케이블 |
| S | 단파장 광섬유 케이블 |
| L | 장파장 광섬유 케이블 |

## 데이터 링크 계층의 이더넷

데이터 링크 계층에서는 이더넷 네트워크에서 주고받는 프레임을 다룬다.

이더넷 프레임은 캡슐화를 거쳐 송신된다.

구성:

- 헤더
- 페이로드
- 트레일러

수신 측에서는 역캡슐화를 거쳐 헤더와 트레일러를 제거한 뒤 상위 계층으로 올려보낸다.

## 이더넷 프레임 구조

| 영역 | 구성 요소 | 설명 |
|---|---|---|
| 헤더 | 프리앰블 | 이더넷 프레임의 시작을 알림 |
| 헤더 | 수신지 MAC 주소 | 데이터를 받을 장치의 MAC 주소 |
| 헤더 | 송신지 MAC 주소 | 데이터를 보낸 장치의 MAC 주소 |
| 헤더 | 타입/길이 | 프레임의 길이 또는 상위 계층 프로토콜 정보 |
| 페이로드 | 데이터 | 상위 계층에서 전달된 내용 |
| 트레일러 | FCS | 오류 검출용 정보 |

## 프리앰블

프리앰블(preamble)은 이더넷 프레임의 시작을 알리는 8바이트(64비트) 크기의 정보이다.

첫 7바이트는 `10101010` 값을 가지고, 마지막 바이트는 `10101011` 값을 가진다.

프리앰블은 송수신지 간 동기화를 위해 사용되며, 실제 송수신 데이터는 아니다.

## MAC 주소

수신지 MAC 주소와 송신지 MAC 주소는 같은 LAN 환경에서 데이터를 정확한 장치로 보내기 위해 사용되는 물리적 주소이다.

MAC 주소는 일반적으로 고유하고, 일반적으로 변경되지 않는 주소이다.

MAC 주소는 네트워크 인터페이스마다 부여되는 6바이트(48비트) 길이의 주소이다.

MAC 주소는 LAN 내에서 송수신지를 특정하는 데 사용된다.

일반적으로 NIC(Network Interface Controller) 장치가 네트워크 인터페이스 역할을 담당한다.

한 컴퓨터에 MAC 주소가 여러 개 있을 수도 있다.

## 타입/길이

타입/길이 필드는 타입 또는 길이를 명시한다.

- 필드에 명시된 크기가 1500 이하이면 프레임의 크기 또는 길이를 의미한다.
- 필드에 명시된 크기가 1536 이상이면 타입을 의미한다.

타입은 이더타입(ethertype)이라고도 한다.

이더타입은 어떤 정보를 캡슐화했는지를 나타내며, 상위 계층에서 사용된 프로토콜이 명시된다.

## 데이터

데이터는 페이로드에 해당한다.

상위 계층에서 전달되거나 전달해야 할 내용이 담긴다.

- 최대 크기: 1500바이트
- 최소 크기: 46바이트

데이터가 46바이트보다 작으면 크기를 맞추기 위한 패딩이 채워진다.

## FCS

FCS(Frame Check Sequence)는 오류 검출용 정보이다.

수신한 이더넷 프레임에 오류가 있는지 확인하기 위한 필드이다.

FCS에는 CRC(Cyclic Redundancy Check)라는 오류 검출용 값이 명시된다.

## 이더넷 외의 기술: 토큰 링

토큰 링 네트워크에서는 호스트들이 링 형태로 연결된다.

호스트들은 돌아가며 토큰이라는 특별한 정보를 교환한다.

네트워크 내 다른 호스트에게 메시지를 송신하려면 반드시 이 토큰을 가지고 있어야 한다.
