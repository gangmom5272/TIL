# 혼자 공부하는 네트워크 1강 ~ 3강

## 컴퓨터 네트워크

컴퓨터 네트워크는 여러 개의 장치가 그물처럼 서로 연결되어 정보를 주고받을 수 있는 통신망이다.

## 인터넷

인터넷은 여러 네트워크를 연결한 네트워크의 네트워크이다.

## 네트워크의 기본 구조

네트워크의 기본 구조는 그래프로 이해할 수 있다.

그래프는 노드와 노드를 연결하는 간선으로 이루어진 자료구조이다.

## 네트워크 구조

- 가장자리에 위치한 노드: 컴퓨터, 휴대폰 같은 호스트
- 중간에 위치한 노드: 공유기 같은 네트워크 장비
- 노드 간 연결: 유선/무선 연결 같은 통신 매체
- 노드 간 주고받는 정보: 데이터 또는 메시지

## 호스트

호스트는 네트워크의 가장자리 노드이다.

네트워크를 통해 주고받는 메시지를 최초로 생성해서 송신하거나, 최종적으로 수신하는 대상이다.

### 호스트의 역할

- 클라이언트: 서버에게 요청을 보내는 호스트
- 서버: 요청에 대한 응답을 보내는 호스트

## 네트워크 장비

네트워크 장비는 호스트 간 주고받는 정보가 거치는 중간 노드이다.

예시:

- 이더넷 허브
- 스위치
- 라우터
- 공유기

네트워크 장비는 호스트 간 주고받는 정보가 수신지까지 안정적이고 안전하게 전송되도록 돕는다.

호스트, 서버, 클라이언트, 네트워크 장비는 구조에 따라 구분한 개념일 뿐, 서로 완전히 배타적인 개념은 아니다.

## 통신 매체

통신 매체는 각 노드를 연결하는 간선 또는 링크이다.

- 유선 매체
- 무선 매체

## 메시지

메시지는 통신 매체로 연결된 노드가 주고받는 정보이다.

예시:

- 웹 페이지
- 파일
- 메일

## 네트워크의 분류

### 범위에 따른 분류

- LAN(Local Area Network)
- WAN(Wide Area Network)

### LAN

LAN은 가까운 지역을 연결한 근거리 통신망이다.

한정된 공간에서 한 개인이나 특정 조직이 구축한 통신망이다.

### WAN

WAN은 먼 지역을 연결하는 광역 통신망이다.

인터넷은 WAN으로 분류할 수 있다.

다른 LAN에 속한 호스트와 메시지를 주고받아야 할 때 WAN이 필요하다.

### ISP

ISP(Internet Service Provider)는 사용자에게 인터넷 같은 WAN에 연결 가능한 회선을 임대하는 등 WAN 관련 서비스를 제공한다.

국내 대표 ISP:

- KT
- LG
- SK

## 메시지 교환 방식에 따른 분류

- 회선 교환 방식
- 패킷 교환 방식

## 회선 교환 방식

회선 교환 방식은 메시지를 주고받기 전에 메시지 전송로인 회선을 설정한 뒤, 해당 회선을 통해 메시지를 주고받는 방식이다.

### 장점

두 호스트 사이에 연결을 확보한 뒤 메시지를 주고받기 때문에, 주어진 시간 동안 전송되는 정보의 양이 비교적 일정하다.

### 단점

회선의 이용 효율이 낮아질 수 있다.

회선만 연결하고 실제 메시지를 주고받지 않으면 회선을 낭비하게 된다.

### 회선 스위치

회선 스위치는 호스트 사이에 일대일 전송로를 확보하는 네트워크 장비이다.

## 패킷 교환 방식

패킷은 패킷 교환 네트워크에서 송수신되는 단위이다.

패킷 교환 방식은 메시지를 패킷이라는 단위로 쪼개어 전송하는 방식이다.

특징:

- 패킷들은 각기 다른 전송로로 이동할 수 있다.
- 패킷의 순서가 수신지에서 뒤바뀔 수 있다.
- 쪼개어 전송된 패킷들은 수신지에서 재조립된다.
- 전송로의 이용 효율이 높다.
- 현대 인터넷은 대부분 패킷 교환 방식을 이용한다.

### 패킷 스위치

패킷 스위치는 패킷의 송수신지를 식별하고, 패킷이 이동할 최적의 경로를 결정하는 장비이다.

예시:

- 스위치
- 라우터

## 패킷의 구조

| 패킷 구성 요소 | 택배 비유 | 의미 |
|---|---|---|
| 페이로드 | 물품 | 전송하려는 데이터 |
| 헤더 | 택배 송장 | 부가 정보 또는 제어 정보 |
| 트레일러 | 택배 송장 | 부가 정보 또는 제어 정보 |
