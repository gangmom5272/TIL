# 데이터 엔지니어의 협업 관계

## 오늘 읽은 내용

견고한 데이터 엔지니어링 Chapter 1에서 데이터 엔지니어가 실무에서 어떤 직군과 함께 일하는지 읽었다.

데이터 엔지니어는 데이터를 직접 생성하는 업스트림 직군과, 데이터를 활용하는 다운스트림 직군 사이에서 데이터가 안정적으로 흐르도록 돕는 역할을 한다.

## 업스트림

업스트림은 데이터가 처음 생성되는 영역이다.

데이터 엔지니어는 업스트림에서 만들어지는 데이터가 안정적으로 수집되고 저장될 수 있도록 협업한다.

### 주요 직군

- 백엔드 엔지니어
- 프론트엔드 엔지니어
- 모바일 앱 개발자
- 서비스 개발자
- 인프라 엔지니어
- DevOps 엔지니어
- 제품 관리자
- 서비스 운영 담당자

### 협업 방식

백엔드 엔지니어와는 서비스 데이터베이스, API, 이벤트 로그 설계에 대해 협업한다.

프론트엔드나 모바일 개발자와는 사용자 행동 로그를 어떤 시점에 어떤 형식으로 남길지 협의한다.

인프라나 DevOps 엔지니어와는 로그 수집, 배포 환경, 모니터링, 장애 대응 방식에 대해 협업한다.

제품 관리자와는 어떤 사용자 행동이나 서비스 지표를 데이터로 남겨야 하는지 논의한다.

업스트림 협업의 핵심은 나중에 분석과 활용이 가능하도록 데이터를 올바른 형태로 생성하고 수집하는 것이다.

## 다운스트림

다운스트림은 수집되고 처리된 데이터를 실제로 사용하는 영역이다.

데이터 엔지니어는 다운스트림 사용자가 필요한 데이터를 신뢰할 수 있는 형태로 사용할 수 있도록 제공한다.

### 주요 직군

- 데이터 분석가
- 데이터 사이언티스트
- 머신러닝 엔지니어
- AI 엔지니어
- 비즈니스 분석가
- 마케팅 담당자
- 기획자
- 경영진
- 제품 관리자

### 협업 방식

데이터 분석가와는 분석에 필요한 테이블, 지표, 데이터 마트, 대시보드용 데이터 구조를 협의한다.

데이터 사이언티스트와 머신러닝 엔지니어와는 모델 학습에 필요한 데이터셋, 피처, 데이터 품질, 데이터 갱신 주기를 협의한다.

비즈니스 분석가나 마케팅 담당자와는 매출, 전환율, 사용자 행동, 캠페인 성과 같은 비즈니스 지표를 제공하기 위해 협업한다.

기획자와 제품 관리자와는 서비스 개선에 필요한 지표와 실험 결과를 확인할 수 있도록 데이터를 제공한다.

경영진에게는 의사결정에 필요한 핵심 지표가 정확하고 일관되게 전달되도록 지원한다.

다운스트림 협업의 핵심은 데이터를 사용하는 사람이 신뢰할 수 있는 데이터를 빠르고 일관되게 사용할 수 있도록 만드는 것이다.

## 데이터 엔지니어의 위치

데이터 엔지니어는 업스트림과 다운스트림 사이에 있다.

업스트림에서 생성된 데이터를 수집하고, 저장하고, 변환하여 다운스트림에서 활용할 수 있는 형태로 제공한다.

즉, 데이터 엔지니어는 데이터 생산자와 데이터 소비자 사이에서 데이터 흐름을 연결하는 역할을 한다.

## 오늘 배운 점

데이터 엔지니어는 혼자 데이터 파이프라인만 만드는 직군이 아니다.

데이터가 생성되는 지점의 개발자, 서비스 담당자와 협업하고, 데이터를 사용하는 분석가, 데이터 사이언티스트, 비즈니스 담당자와도 협업한다.

데이터 엔지니어링은 기술적인 파이프라인 구축뿐 아니라, 조직 안에서 데이터가 올바르게 생성되고 활용되도록 연결하는 일이다.
