## 데이터 엔지니어링

데이터 엔지니어링은 데이터를 수집, 저장, 처리, 제공할 수 있는 기반을 설계하고 운영하는 활동이다.

데이터가 분석, 서비스, 의사결정에 활용될 수 있도록 데이터 흐름을 만드는 일이 핵심이다.

## 데이터 엔지니어

데이터 엔지니어는 데이터가 필요한 곳에 안정적으로 도달할 수 있도록 데이터 파이프라인과 저장 구조를 설계하고 관리하는 역할을 한다.

단순히 데이터를 옮기는 사람이 아니라, 데이터의 품질, 신뢰성, 활용 가능성을 높이는 역할을 한다.

## 데이터 엔지니어링 수명 주기

데이터 엔지니어링 수명 주기는 데이터가 발생해서 최종적으로 활용되기까지의 전체 흐름을 의미한다.

데이터 생성, 수집, 저장, 변환, 제공, 활용 같은 단계로 이해할 수 있다.

## 데이터 성숙도

데이터 성숙도는 조직이 데이터를 얼마나 잘 수집, 관리, 활용하고 있는지를 나타내는 수준이다.

데이터 성숙도가 낮은 조직에서는 기본적인 데이터 수집과 저장 기반이 중요하고, 성숙도가 높은 조직에서는 데이터 품질, 자동화, 거버넌스, 확장성 같은 요소가 더 중요해진다.
